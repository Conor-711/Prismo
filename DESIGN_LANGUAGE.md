# DESIGN_LANGUAGE.md — Prismo 设计语言

> 最近更新：2026-06-22
> 配套实现：`web/tailwind.config.ts`（颜色/圆角/字体 token）、`web/app/globals.css`（`:root` CSS 变量，**暗色单主题**）。
> 本文件是设计决策的「宪法」。改 token 前先读这里；改完 token 也要回来同步本文件。

**一句话内核：** 信号优先的黑色终端 —— 黑底=静默与专注，青绿=从多方噪音里被**提炼出的那一个信号**，留白=焦点。把五地噪音收敛成一束清晰的绿。

---

## 1 · 五条原则
1. **黑为主，绿为辅，克制用色** —— 一屏青绿覆盖面积 < ~10%，只给「信号」。
2. **数据是主角** —— 关键数字放大，装饰退场；每个模块只保留一个信息焦点。
3. **扁平不发光** —— 靠发丝边 + 留白分层，不靠阴影/渐变填充/光晕（反 AI 味）。
4. **语义全站统一** —— 多空色不随地区红绿翻转（见 §3）。
5. **动效服务理解** —— 动画只用于「引导视线 / 表达数据变化」，绝不装饰。

## 2 · 色彩（暗色单主题，已停用白天模式）
| 角色 | Tailwind 类 | CSS 变量 | 值 | 用途 |
|---|---|---|---|---|
| 页面底 | `bg-ink` | `--c-ink` | `#121212` | 画布，近黑中性 |
| 卡片/顶栏 | `bg-card`/`bg-surface` | `--c-card`/`--c-surface` | `#161616` | 模块基底 |
| 抬升/悬停 | `bg-elevated` | `--c-elevated` | `#1C1D1F` | hover 态 |
| 发丝边 | `border-line` | `--c-line` | `#2A2D2F` | 唯一的分层手段 |
| 主文字 | `text-cream` | `--c-cream` | `#F1F3F4` | 标题/正文 |
| 次文字 | `text-neutral-400` | `--n-400` | `#A7A9AC` | 标签/说明 |
| 弱文字 | `text-neutral-600` | `--n-600` | `#6B6D70` | 元信息/时间 |
| **品牌青绿** | `text-brand`/`bg-brand` | 固定 | `#57D7BA` | 链接·选中·主 CTA·正向·焦点描边·信号高亮 |
| 看涨 bull | `text-bull`/`bg-bull` | 固定 | `#57D7BA` | = 品牌绿 |
| 看跌 bear | `text-bear`/`bg-bear` | 固定 | `#FF5C6C`（珊瑚） | 看跌/负向情绪 |
| 异动/高信号 | `text-gold` | `--c-gold` | `#F7D14E` | 高热度/异动旗标，**极克制** |
| 中性情绪 | `text-neutral-500` | `--n-500` | `#848689` | 无明显多空 |

**⚠ 遗留别名（沿用旧类名免大改，含义已变）：** `reddit` / `amber` / `brand` 三个 Tailwind 颜色类**都 = 青绿 `#57D7BA`**；`amber` 不是琥珀色（真正的琥珀/金信号用 `gold`）。图表里的发散绿用 `#57D7BA`、发散红用 `#FF5C6C`、中性灰用 `#7A8A96`/`#8A8A93`。

**用色纪律（「黑为主绿为辅」的关键）：** 青绿**只**出现在 链接 / 选中态 / 主按钮 / 正向数值 / 焦点描边 / 关键信号；**禁止**用作大面积填充或背景。整套调色板压到 **4 个角色**：黑灰阶 + 青绿 + 珊瑚 + 琥珀。

**品牌渐变 `--grad-brand`：** `青绿 #57D7BA → 深松绿 #2F6F63`（**去紫**，保持黑+绿；`.btn-grad` 描边同源）。

## 3 · 多空语义 vs 地区身份（解决红绿文化冲突）
- **多空语义全站固定：** 看涨永远青绿 `#57D7BA`、看跌永远珊瑚 `#FF5C6C`，**不随 美/中/日/韩/台 翻转**（美股绿涨、亚洲红涨的冲突由此规避）。
- **地区身份色另起一套：** 五地区用各自中性化的**身份色**做识别，与「涨跌」语义解耦——避免「红」既是地区色又是涨跌色的歧义。
- 强弱用**同色不同透明度**表达（如青绿 28%→100%），不引入新色相。

## 4 · 字体（几何无衬线 · 现代）
- **标题/UI = Figtree**（几何人文无衬线，`--font-sora`/`--font-inter`）：700–800 紧字距做 hero/H1，600 做小标题。
- **数据/数字 = Roboto + `tabular-nums`**（`--font-mono`，工具类 `.tabular`）：所有数字等宽对齐，KPI 28–40px / 600。
- 字阶：Hero 64–88 / H1 24–28 / H2 18–20 / 正文 14–15 / 眉标 11–12（`.eyebrow`，大写 + 字距 0.14em）。
- 不引入衬线、不引入第三字族（保持现代/科技/专业）。

## 5 · 间距与密度（均衡 · 适度留白）
- 内容最大宽 **1480px**（应用主区 `max-w-[1480px]`）；落地页 1180px。
- 卡片内边距 16–20px；模块纵向间距 16–24px（`space-y-4/6`）；栅格间距 16px。
- 重点数据区给更多留白，次要信息收进折叠/次级层级 —— **一屏一焦点**。

## 6 · 形状与质感
- 圆角扁平：卡片/输入 **2–4px**（`rounded`/`rounded-lg`），按钮/徽章 **胶囊**（`rounded-full`）；无大圆角。
- 分层只靠 `--panel-shadow`（`inset 0 0 0 1px #2a2d2f` 发丝边 + 极淡投影），不堆叠重阴影。
- hover 用 `.panel-hover`：`translateY(-1px)` + 青绿描边（`--panel-shadow-hover`）。
- 胶片颗粒 `--grain-opacity: 0`（默认关闭）；无光晕、无渐变大色块。

## 7 · 数据可视化（大方 · 有焦点）
- **极简线性：** 细线、最少网格线、无边框填充；坐标轴弱化为 `#7A8A96`。
- 严格走语义色：涨 `#57D7BA` / 跌 `#FF5C6C` / 中性灰；发散刻度 = 珊瑚 ← 灰 → 青绿。
- 每图一个**放大的焦点数字 + sparkline**，细节按需展开。
- 热力/对比用透明度梯度，不用彩虹色。图表 tooltip：底 `#161616`、边 `#2a2d2f`。
- ECharts 等 canvas 图为暗色单主题，配色硬取暗色（`lib/useTheme.ts` 的 `useIsLight()` 恒 `false`）。

## 8 · 动效（适度 · 有引导）
- 微交互（hover/切换）：**120–160ms** ease-out。
- 入场（卡片/图表）：**200–280ms**，`translateY 4–8px + fade`，stagger 30–50ms（参考 `.onb-in` 的 `cubic-bezier(.22,1,.36,1)`）。
- 数字滚动：复用 `Counter` 组件，**600–900ms** ease-out。
- 图表 draw-in：300–500ms。
- **硬约束：** 无循环/呼吸/发光动画（信号条 `.tape` 跑马灯除外）；`prefers-reduced-motion` 一律降级。

## 9 · 组件语气
- 按钮：主 = 胶囊青绿实底；次 = 青绿描边；`.btn-grad`（青绿→深松绿渐变描边）只留给**唯一关键入口**，不滥用。
- 徽章/标签：发丝边 + 弱底，语义色仅染文字 / 小圆点。
- 表格：tabular 数字右对齐，行 hover `bg-elevated`，无斑马纹。
